# Projects

This section contains Systems Engineering projects — both personal study sandboxes and sanitized real-world work.

---

## Project Types

| Folder | Description |
|--------|-------------|
| [`_project-template/`](./_project-template/) | **Start here.** Copy this scaffold for every new project. |
| [`Study-Projects/`](./Study-Projects/) | Practice case studies and CSEP exam scenarios |
| [`Real-World-Projects/`](./Real-World-Projects/) | Sanitized real-world project artifacts (no proprietary data) |

---

## How to Start a New Project

1. Copy the `_project-template/` folder to the appropriate category
2. Rename it: `[YYYY]-[ProjectName]` (e.g., `2026-SmallSat-ConOps`)
3. Update the project `README.md` inside the folder
4. Begin populating the subfolders

---

## Project Template Structure

```
_project-template/
├── README.md              ← Project overview, objectives, status
├── docs/                  ← ConOps, SRS, SDD, T&EP, ICD
├── models/                ← MBSE models (SysML, Capella, etc.)
├── requirements/          ← Requirements files and RTM
├── architecture/          ← Architecture artifacts, views, diagrams
├── verification-validation/ ← Test plans, V&V matrix, results
└── risk-register/         ← Risk register and risk management plan
```
